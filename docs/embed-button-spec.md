# 応援会員ボタン（外部設置）仕様 — 紙のみ / 未実装

ステータス: **設計のみ。コードフリーズ中につき実装しない。** Day30ゲート通過後の実装候補。
動機: 店主が常に人に会えるわけではない。店頭POP（同期・対面）を補う「非同期の接点」を、店が既に持つWeb上に置く。
最終更新: 2026-06-19

---

## 0. 何のための機能か（壊れた前提を先に潰す）

- これは「自社サイトにJSを貼る本格Widget」が主役**ではない**。文化拠点の大半は自前サイトを持たない／持っても月間アクセスは二桁。
- 主役は **(A) どこにでも置ける1本のリンク＋ボタン画像**。Instagramプロフィール・note・LINE公式・メール署名・QR — 客が実際にいる場所に置く。
- **(B) JS埋め込み** は「自社サイトを持つ少数」向けのオマケ。実装の重心はここに置かない。
- 価値の本体は新規開発ではなく **着地先（既存の /join）と計測（?ref=）**。新規APIは原則ゼロ。

## 1. 着地先（既存・変更最小）

`https://<host>/spots/{spotId}/join?plan={amount}&ref={channel}`

- `/spots/[spotId]/join` は実装済み。`?plan=` も既に受ける（300/1000）。
- 追加するのは `?ref=` の解釈のみ（後述）。ページ自体の改修は不要。

## 2. 配布形態

### A層: リンク＋ボタン画像（全店向け・これが本命）
店主に渡すのはコピペ用の1行と、画像URL1本。

```html
<a href="https://<host>/spots/{spotId}/join?ref=web">
  <img src="https://<host>/badge/{spotId}.svg" alt="この店の応援会員になる" height="48">
</a>
```

- `badge/{spotId}.svg` = ホスト配信の静的バッジ（店名 or 共通ロゴ）。サーバー処理なしの静的アセット、もしくは軽い動的SVG。
- Instagram bio など `<a>` すら置けない場所向けには、**素のURLだけ**渡せば成立する（ボタンは諦める）。これが最小形。

### B層: JS埋め込み（自社サイト持ちの少数向け・オマケ）
```html
<script src="https://<host>/embed.js" data-spot="{spotId}" data-plan="500"></script>
```
- `embed.js` は静的JS。`data-spot` を読み、その場に A層と同じ `<a><img></a>` を描画するだけ。
- iframe決済は**やらない**（Stripe Checkoutは別タブ遷移で十分／PCI・CSP・高さ問題を避ける）。クリックで `/join` へ遷移。
- **サーバー実装ゼロ**。静的配信のみ。

## 3. 計測（この機能の本当の取り分）

`?ref=` でチャネルを識別し、会員化まで貫通させる。

- 値の例: `pop`（店頭POP） / `qr` / `web`（A層リンク） / `embed` / `ig`（Instagram） / `mail`。
- フロー: `/join` が `ref` を読む → Checkout作成時に **Stripe subscription の metadata に `ref` を載せる**（[app/api/stripe/checkout/route.ts](app/api/stripe/checkout/route.ts) で metadata 付与）。
- これにより「どのチャネルが会員を生んだか」が後から集計できる。
  - **H1検証**: pop（対面） vs web/embed（非同期）の転換率比較。対面が圧倒的なら embed は捨て、POPに全振りの判断材料になる。
  - **H4検証**: SNS経由（ig）のシェア→加入率。

> 計測がないと、この機能は「作った気」で終わる。`?ref=` だけは実装時に必ず入れる。

## 4. やらないこと（スコープの壁）

- iframe内決済 / ポップアップ決済モーダル → やらない。別タブ遷移で十分。
- 店ごとのボタンデザイン編集UI → やらない。バッジは1〜2種固定。
- 外部サイトの会員状態取得API（「この訪問者は会員か」を外部に返す）→ やらない。越権・個人情報・保守地獄。
- OGP動的生成、A/Bテスト基盤 → ゲート後の後の後。

## 5. 実装したくなった時の最小手順（ゲート通過後）

1. `/join` で `?ref=` を読み、Checkout metadata に載せる（既存ファイル1つの改修）。
2. 静的バッジSVGを `public/badge/` か軽い動的ルートで配信。
3. 営業キット（[docs/sales/](sales/)）に「Webにも貼れます」の1枚を追加し、コピペ用スニペットを同梱。
4. （任意）`embed.js` 静的配信。ここは需要が出てから。

工数の体感: 1 が本体で半日、2 が半日、3 が紙仕事。B層は後回しでよい。

## 6. 今すぐ営業に使う（コード不要・本ドキュメントの主目的）

- 商談やDMで「店頭だけでなく、お店のInstagramやnote、メールの署名にも"応援会員になる"リンクを置けます」と言える。
- 現状でも **共有パネル（[components/owner/spot-share-panel](../components/owner/spot-share-panel.tsx) 相当）が QR＋リンクを既に出している**ので、「リンクは今日から配れる」は嘘ではない。バッジ画像が無いだけ。
- → 対面できない相手にも「URL1本」で接点を作れる、というのが非同期チャネルの最小実装。**これは今日から運用可能。**
